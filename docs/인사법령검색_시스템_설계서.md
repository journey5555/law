# 인사법령 검색 서비스 시스템 설계서

---

## 1. 프로젝트 개요

### 1.1 서비스 목적

인사 담당자 및 법무 실무자가 노무 관계 법령과 판례를 신속하고 정확하게 검색·조회하고, 법령 개정 시 자동으로 변경 내용을 인지할 수 있는 AI 기반 인사법령 검색 서비스.

### 1.2 요구사항

| 구분 | 내용 |
|------|------|
| 자동 수집 | 국가법령정보센터 연동 자동 수집 스케줄러 구현 |
| 법령 검색 | 노무 관계 법령(근로기준법, 남녀고용평등법, 산업안전보건법, 상법 등) 검색 및 조회 인프라 구축 |
| 판례 검색 | 관련 판례 검색 및 법령·판례 연계 조회 기능 구현 |
| 개정 모니터링 | 법령 개정사항 주기별 자동 모니터링 및 변경 알림 기능 구현 |
| 수집 관리 화면 | 법령별 수집 주기 설정, 수동 실행, 수집 이력 로그, 실패 알림 관리 화면 구현 |

---

## 2. 시스템 아키텍처

### 2.1 데이터 인제스션 파이프라인 (배치/스케줄)

외부 소스에서 법령·판례 데이터를 수집하여 A.X Knowledge에 적재하는 오프라인 흐름.

```
     [국가법령정보센터 API]
                │
                ▼
   [자동 수집 Agent (Scheduler)]
                │
     ┌──────────┴──────────┐
     ▼                     ▼
[법령 수집]            [판례 수집]
     │                     │
     ▼                     ▼
[A.X Knowledge]    [A.X Knowledge]
  (법령 Repository)  (판례 Repository)
```

> 개정 모니터링: 자동 수집 Agent가 주기적으로 변경분을 탐지하여 Vector DB Upsert 및 변경 알림 전달

---

### 2.2 검색 요청 처리 파이프라인 (실시간)

사용자 질의가 들어왔을 때 Agent가 처리하여 응답을 반환하는 온라인 흐름.

```
              [UI]
               │
               ▼
          [SI 시스템]
               │
    ┌──────────┴──────────┐
    │ (법령 질의)           │ (판례 질의)
    ▼                     ▼
[법령 Agent]          [판례 Agent]
    │                     │
    ├ 1. Query Rewriting   ├ 1. Query Rewriting
    ├ 2. 법령 Retriever    ├ 2. 판례 Retriever
    ├ 3. Generator         ├ 3. 법령 Agent 연계 호출
    │                     │    (참조조문 조회)
    │                     ├ 4. Generator
    │                     │    (판례 + 조문 Context 결합)
    ▼                     ▼
  [응답]               [응답]
    └──────────┬──────────┘
               ▼
          [SI 시스템]
               │
               ▼
             [UI]
```

---

### 2.3 컴포넌트 역할

| 컴포넌트 | 역할 |
|----------|------|
| 자동 수집 Agent | 국가법령정보센터 데이터 주기적 수집 및 개정 모니터링 |
| A.X Knowledge | 수집 데이터 Embedding 및 Vector DB 인제스션 |
| 법령 Agent | 법령 조문 검색 및 해석 전용 에이전트 |
| 판례 Agent | 판례 검색·요약 및 법령 Agent 연계 전용 에이전트 |
| Agent API | SI 시스템과 에이전트 간 인터페이스 |
| SI 시스템 | Agent API 호출 및 UI 응답 전달 |

## 3. 데이터 인제스션 파이프라인

### 3.1 수집 소스 및 대상

| 소스 | 수집 대상 | 주요 수집 항목 |
|------|----------|---------------|
| 국가법령정보센터 API | 현행 법령 (`target=law`) | 법령명, 소관부처, 시행일, 조문(조·항·호 계층) |
| 국가법령정보센터 API | 판례 (`target=prec`) | 사건번호, 법원명, 선고일자, 판시사항, 판결요지, 참조조문, 판례내용 |
**수집 대상 법령 범위**: 근로기준법, 남녀고용평등법, 산업안전보건법, 상법 등 노무 관계 법령

### 3.2 인제스션 흐름

```
[국가법령정보센터 API]
        │
        ▼
[데이터 수집 및 파일 변환]
 - 법령: 조문 단위 분리 (조·항·호 계층 유지)
 - 판례: 판시사항·판결요지·참조조문 메타데이터 추출
        │
        ▼
[1단계] POST /api/v1/datasources/upload/files
 - 필드명: files (multipart/form-data)
 - 응답: temp_file_path 반환
        │
        ▼
[2단계] POST /api/v1/knowledge/repos/{repo_id}/add_document_and_indexing
 - 요청: { "document_file_info": { "temp_file_path": "...", "file_name": "..." } }
 - 파일 추가 및 인덱싱 동시 실행
        │
        ▼
[A.X Knowledge Repository (법령 / 판례 분리)]
```

**인제스션 설정 항목** (Repository 생성 시 지정)

| 항목 | 설명 |
|------|------|
| `embedding_model_name` | 사용할 Embedding 모델 |
| `chunk_size` | Chunk 최대 토큰 수 |
| `chunk_overlap` | Chunk 간 겹침 토큰 수 |
| Splitter 방식 | RecursiveCharacter / Semantic / Custom 중 선택 |
| Retrieval 모드 | dense / sparse / hybrid / semantic 중 선택 |

### 3.3 개정 모니터링 및 업데이트 관리

법령은 제·개정이 빈번하게 발생하므로 아래 프로세스로 자동 관리한다.

| 단계 | 내용 |
|------|------|
| 주기적 탐지 | 스케줄러(CronJob)가 국가법령정보센터 API를 주기적으로 호출하여 공포일자·시행일자 변경 여부 확인 |
| 변경 탐지 | 기존 수집 데이터와 비교하여 신규·개정·폐지 법령 식별 |
| Vector DB 갱신 | 변경된 조문만 부분 업데이트(Upsert)하여 최신 상태 유지 |
| 변경 알림 | 개정 내용을 SI 시스템으로 전달하여 사용자에게 변경 알림 제공 |

## 4. 에이전트 설계

### 4.1 법령 Agent

**역할**: 법령 조문 검색 및 해석

| 단계 | 내용 |
|------|------|
| Query Rewriter | 구어체·약어를 법률 용어로 변환, 핵심 조문 키워드 추출 |
| Retriever | Vector DB 법령 컬렉션 검색 (법령명·조번호 필터 지원) |
| Generator | 검색된 조문 Context 기반 해석 생성, 조문 외 내용 생성 금지 |

**출력 형식**
```
- 법령명 및 조번호
- 조문 원문
- 해석 요약
```

### 4.2 판례 Agent

**역할**: 판례 검색, 요약 및 참조 법령 연계

| 단계 | 내용 |
|------|------|
| Query Rewriter | 사실관계 기반 판례 검색에 최적화된 쿼리 확장 |
| Retriever | Vector DB 판례 컬렉션 검색 (법원·선고일 필터 지원) |
| 법령 Agent 연계 | 검색된 판례의 참조조문 추출 → 법령 Agent 호출 → 조문 원문 결합 |
| Generator | 판례 Context + 연계된 조문 Context 기반 요약 생성 |

**출력 형식**
```
- 사건명 및 사건번호
- 사건 개요 (사실관계 요약)
- 법원의 판단 (핵심 판단 이유)
- 참조 법령 (조문 원문 연계)
```

### 4.3 에이전트 간 연계 흐름

```
사용자 질의
    │
    ▼
[판례 Agent]
    │
    ├─ 1. Query Rewriting
    ├─ 2. 판례 Retriever (판례 컬렉션 검색)
    ├─ 3. 검색된 판례에서 참조조문 파싱
    │       예: "민법 제750조, 근로기준법 제23조"
    │
    ├─ 4. 법령 Agent API 호출
    │       요청: { "references": ["민법 제750조", "근로기준법 제23조"] }
    │       응답: 각 조문 원문 텍스트
    │
    └─ 5. Generator
            입력: 판례 Context + 조문 Context
            출력: 판례 요약 + 참조 조문 내용 포함 응답
```

## 5. API 인터페이스

### 5.1 SI → Agent API (SKT AIP Agent Gateway)

**Base URL**: `https://aip.sktai.io/api/v1/agent_gateway`

| 방식 | 엔드포인트 | 설명 |
|------|-----------|------|
| 동기 | `POST /{AGENT_ID}/invoke` | 응답 완료 후 전체 반환 |
| 스트리밍 | `POST /{AGENT_ID}/stream` | 토큰 단위 실시간 스트리밍 (SSE) |

**요청 구조**
```json
{
  "input": {
    "messages": [
      { "type": "human", "content": "부당해고 위로금 산정 기준 판례 알려줘" }
    ]
  }
}
```

**인증**: `Authorization: Bearer {AGENT_API_KEY}`

**응답 구조 (invoke)**
```json
{
  "output": {
    "content": "생성된 답변 텍스트"
  },
  "run_id": "..."
}
```

**응답 구조 (stream)**
```
data: {"final_result": "토큰..."}
data: {"final_result": "토큰..."}
data: [DONE]
```

> 법령 Agent와 판례 Agent는 각각 별도의 `AGENT_ID`로 배포되며, SI에서 질의 유형에 따라 호출할 Agent를 선택한다.

---

### 5.2 자동 수집 Agent → DataSource / Knowledge API

**인증**: OAuth2 JWT Bearer Token (Keycloak 발급)

#### Step 1 — 파일 업로드

| 메서드 | 엔드포인트 |
|--------|-----------|
| `POST` | `https://aip.sktai.io/api/v1/datasources/upload/files` |

```
요청: multipart/form-data, 필드명 files
응답: { "data": [{ "temp_file_path": "/data/data/datasource/temp/...", "file_name": "..." }] }
```

#### Step 2 — 인덱싱 실행

**Base URL**: `https://aip.sktai.io/api/v1/knowledge`

| 메서드 | 엔드포인트 | 설명 |
|--------|-----------|------|
| `POST` | `/repos/{repo_id}/add_document_and_indexing` | 파일 추가 및 인덱싱 동시 실행 |
| `POST` | `/repos/{repo_id}/indexing` | 기존 등록 파일 전체 재인덱싱 |
| `DELETE` | `/repos/{repo_id}/documents` | 폐지 법령 문서 삭제 |

```json
{
  "document_file_info": {
    "temp_file_path": "/data/data/datasource/temp/2e97fe7b-c9f_20260609100621191126.md",
    "file_name": "근로기준법.md"
  },
  "document_metadata": {
    "category": "노무법령",
    "tags": ["근로기준법", "인사"]
  }
}
